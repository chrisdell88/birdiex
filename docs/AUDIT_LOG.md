### Findings

- **[P2] `npm run lint` is not in the deploy gate → 12 lint errors have piled up invisibly.**
  `npm run build` runs the 6 verify guards + vitest + `tsc -b` + `tsc --noEmit` +
  `vite build`, but never `eslint`. So every Vercel deploy stays green while lint
  errors accumulate. The bot commits (auto-roll, ticker, live-refresh) also never
  run lint. CLAUDE.md "Before Shipping Any Change" rule #2 says lint must pass —
  but nothing enforces it. Current `npx eslint .` = 12 errors, 1 warning:
  - `scripts/auto-roll.ts:387,388,389,392,396` — `no-regex-spaces`. These are the
    YAML-indent regexes (`/^  SLUG:/m`) that patch the workflow files on event
    switch. They are FUNCTIONALLY CORRECT (two literal spaces match the 2-space
    YAML indent) and `verify:auto-roll` passes — eslint just dislikes literal
    double-spaces. Fix is cosmetic: `/^ {2}SLUG:/m` or an eslint-disable line.
  - `scripts/build-event.ts:190` — `runCount` assigned but never used.
  - `scripts/build-outrights.ts:13,15,129` — `existsSync`, `pathToFileURL`,
    `outPath` imported/assigned but never used (leftovers from the no-merge
    refactor on 2026-06-07).
  - `scripts/verify-rankings-data-completeness.ts:23` — `readFileSync` unused.
  - `src/components/BacktestLab.tsx:378` — empty block statement; `:386` —
    `setState` called synchronously inside a `useEffect` (react-hooks rule). Live
    code (lazy-imported in App.tsx), not dead.
  - `src/components/RankingsTable.tsx:253` — `useMemo` missing dep `lastRoundField`
    (warning, not error).
  Why it matters: the rule Chris wrote ("lint must pass") is silently a no-op.
  Either wire `npm run lint` into the build or downgrade the rule in CLAUDE.md so
  it isn't a false promise. Suggested fix: add `&& npm run lint` to the `build`
  script after the verify guards (after cleaning the 11 real errors).

- **[P2] Stale comments in `src/config/event.ts:11-16` describe the wrong round.**
